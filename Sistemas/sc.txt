C:\>sc

DESCRIPTION:
        SC is a command line program used for communicating with the
        Service Control Manager and services.
USAGE:
        sc <server> [command] [service name] <option1> <option2>...

        The option <server> has the form "\\ServerName"
        Further help on commands can be obtained by typing: "sc [command]"
        Commands:
          query-----------Queries the status for a service, or												control---------Sends a control to a service.
                          enumerates the status for types of services.										sdshow----------Displays a service's security descriptor.
          queryex---------Queries the extended status for a service, or										sdset-----------Sets a service's security descriptor.
                          enumerates the status for types of services.										showsid---------Displays the service SID string corresponding to an arbitrary name.
          start-----------Starts a service.																	triggerinfo-----Configures the trigger parameters of a service.
          pause-----------Sends a PAUSE control request to a service.										preferrednode---Sets the preferred NUMA node of a service.
          interrogate-----Sends an INTERROGATE control request to a service.								GetDisplayName--Gets the DisplayName for a service.
          continue--------Sends a CONTINUE control request to a service.									GetKeyName------Gets the ServiceKeyName for a service.
          stop------------Sends a STOP request to a service.												EnumDepend------Enumerates Service Dependencies.
          config----------Changes the configuration of a service (persistent).								qc--------------Queries the configuration information for a service.
          description-----Changes the description of a service.												qdescription----Queries the description for a service.
          failure---------Changes the actions taken by a service upon failure.								qfailure--------Queries the actions taken by a service upon failure.
          failureflag-----Changes the failure actions flag of a service.									qfailureflag----Queries the failure actions flag of a service.
          sidtype---------Changes the service SID type of a service.										qsidtype--------Queries the service SID type of a service.
          privs-----------Changes the required privileges of a service.										qprivs----------Queries the required privileges of a service.
          managedaccount--Changes the service to mark the service account									qtriggerinfo----Queries the trigger parameters of a service.
                          password as managed by LSA.														qpreferrednode--Queries the preferred NUMA node of a service.
		  qmanagedaccount-Queries whether a services uses an account with a									qprotection-----Queries the process protection level of a service.
						  password managed by LSA.															delete----------Deletes a service (from the registry).
          quserservice----Queries for a local instance of a user service template.							create----------Creates a service. (adds it to the registry).
		  
        The following commands don't require a service name:
        sc <server> <command> <option>
          boot------------(ok | bad) Indicates whether the last boot should
                          be saved as the last-known-good boot configuration
          Lock------------Locks the Service Database
          QueryLock-------Queries the LockStatus for the SCManager Database

EXAMPLE:
        sc start MyService

QUERY and QUERYEX OPTIONS:																					SYNTAX EXAMPLES
        If the query command is followed by a service name, the status										sc query                - Enumerates status for active services & drivers
        for that service is returned.  Further options do not apply in										sc query eventlog       - Displays status for the eventlog service
        this case.  If the query command is followed by nothing or one of									sc queryex eventlog     - Displays extended status for the eventlog service
        the options listed below, the services are enumerated.												sc query type= driver   - Enumerates only active drivers
    type=    Type of services to enumerate (driver, service, userservice, all)								sc query type= service  - Enumerates only Win32 services
             (default = service)																			sc query state= all     - Enumerates all services & drivers
    state=   State of services to enumerate (inactive, all)													sc query bufsize= 50    - Enumerates with a 50 byte buffer
             (default = active)																				sc query ri= 14         - Enumerates with resume index = 14
    bufsize= The size (in bytes) of the enumeration buffer													sc queryex group= ""    - Enumerates active services not in a group
             (default = 4096)																				sc query type= interact - Enumerates all interactive services
    ri=      The resume index number at which to begin the enumeration										sc query type= driver group= NDIS     - Enumerates all NDIS drivers
             (default = 0)
    group=   Service group to enumerate
             (default = all groups)
